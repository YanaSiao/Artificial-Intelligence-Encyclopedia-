## Fundamental Difference: On-Policy vs. Off-Policy

The core distinction lies in the **Target** used for the update.

- **SARSA (On-Policy):** Updates $Q(S_t, A_t)$ based on the action $A_{t+1}$ actually chosen by the current policy (e.g., $\epsilon$-greedy). It learns the value of the policy it is actually following, including its exploratory moves.
    
- **Q-Learning (Off-Policy):** Updates $Q(S_t, A_t)$ based on the maximum possible $Q$-value of the next state ($max_{a'} Q(S_{t+1}, a')$). It learns the value of the **optimal** policy, regardless of the exploratory actions taken by the behavior policy.
---
## Convergence and optimality

|**Feature**|**SARSA**|**Q-Learning**|
|---|---|---|
|**Convergence Condition**|**GLIE** (Greedy in the Limit with Infinite Exploration) + Robbins-Monro step-size.|All state-action pairs $(s,a)$ must be visited infinitely often + Robbins-Monro.|
|**Optimality**|Converges to $Q^*$ only if $\epsilon$ is decayed to zero.|Converges to $Q^*$ even if $\epsilon$ remains constant (as long as all pairs are visited).|
|**Stability**|Generally more stable in the presence of function approximation.|Can be unstable (the "Deadly Triad": Off-policy + Bootstrapping + Function Approximation).|

---
## Behavior and Performance: The "Cliff Walking" Case

A classic way to compare these two is the **Cliff Walking** environment.

- **SARSA** learns a "safe" path. Because it is on-policy, it realizes that it might occasionally fall off the cliff due to its own $\epsilon$-greedy exploration. It chooses a path far from the edge to maximize the expected return of its actual (exploratory) behavior.
    
- **Q-Learning** learns the "optimal" path. It learns the values for the path right along the edge of the cliff because it assumes it will act greedily in the future. However, during training, it will frequently fall off the cliff because its behavior policy is still $\epsilon$-greedy.
---
##  Complexity Analysis

Both algorithms are highly efficient in their tabular forms:

- **Time Complexity:** Both are **$O(1)$** per time step. They involve a single table lookup, a simple arithmetic update, and (for Q-learning) a `max` operation over a finite action set $\mathcal{A}$.
    
- **Space Complexity:** Both require **$O(|\mathcal{S}| \times |\mathcal{A}|)$** to store the $Q$-table.
    
- **Sample Complexity:** Q-learning is generally considered more sample-efficient because it can re-use data from any behavior policy (off-policy), whereas SARSA requires data generated by the current policy.

## Summary Comparison Table

|**Feature**|**SARSA**|**Q-Learning**|
|---|---|---|
|**Update Rule**|$Q(s,a) \leftarrow Q + \alpha [R + \gamma Q(s', a') - Q]$|$Q(s,a) \leftarrow Q + \alpha [R + \gamma \max_{a'} Q(s', a') - Q]$|
|**Policy Type**|On-Policy|Off-Policy|
|**Action choice**|Uses actual next action $a'$|Uses hypothetical best action $a'$|
|**Risk Profile**|Conservative (considers exploration risk)|Radical (assumes optimal future behavior)|
|**Primary Use**|When the cost of "online" errors is high.|When learning from batches or historical data.|
